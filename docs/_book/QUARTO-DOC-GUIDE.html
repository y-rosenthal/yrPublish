4  QUARTO-DOC-GUIDE.qmd

This document explains how to build and regenerate the HTML documentation book for this project. The book is built with Quarto and turns the markdown files in docs/ into a single navigable HTML site.

4.1 What gets built

  • Source: All .qmd files in docs/ that are listed in docs/_quarto.yml.
  • Output: Rendered HTML and assets go into docs/_book/. Open docs/_book/index.html in a browser to read the book.

4.2 When to regenerate

Regenerate the book whenever you:

  • Add, remove, or rename documentation files (and update _quarto.yml if needed).
  • Edit any of the docs that are included in the book.

4.3 Installing Quarto

You need the Quarto CLI to render the book. If quarto --version works in your terminal, you can skip this section.

4.3.1 Option A: Official installer (recommended)

  1. Go to https://quarto.org/docs/get-started/.
  2. Choose your OS and follow the install steps. On Linux you can use the .deb package or the tarball; on macOS, the .pkg; on Windows, the .msi.

4.3.2 Option B: Linux (tarball, single user)

Replace VERSION with a release from Quarto releases (e.g. 1.4.553):

# Download and extract
wget https://github.com/quarto-dev/quarto-cli/releases/download/v${VERSION}/quarto-${VERSION}-linux-amd64.tar.gz
mkdir -p ~/opt
tar -C ~/opt -xzf quarto-${VERSION}-linux-amd64.tar.gz

# Make quarto available (adjust if you use a different bin directory)
mkdir -p ~/.local/bin
ln -sf ~/opt/quarto-${VERSION}/bin/quarto ~/.local/bin/quarto

# Ensure ~/.local/bin is in PATH (e.g. add to ~/.bashrc or ~/.profile)
export PATH="$HOME/.local/bin:$PATH"

4.3.3 Verify

quarto check

This checks the CLI and suggests any missing tools (e.g. Pandoc).

4.4 Regenerating the book

From inside the docs/ directory (recommended):

cd docs
quarto render

Or from the project root (the directory that contains docs/):

quarto render docs

If you see errors about missing environment variables when running from the project root, run the command from inside docs/ instead so only the book project is used.

After a successful run, open docs/_book/index.html in your browser to view the book.

4.4.1 Optional: preview while editing

To auto-rebuild when files change and optionally open the book in a browser:

quarto preview docs

Press Ctrl+C to stop the preview server.

4.5 Adding or removing chapters

The book’s table of contents (TOC) on the left is defined in docs/_quarto.yml under book.chapters. Use the steps below so that TOC entries show the actual filenames as links and, when you have multiple versions of a doc, the most recent appears first.

4.5.1 How to add a new doc

  1. Add the .qmd file (e.g. My-New-Doc.qmd) to docs/.

  2. Set the document’s YAML title to the filename (including .qmd). That way the left-margin TOC will use the filename as the link text:

    ---
title: "My-New-Doc.qmd"
---

  3. In docs/_quarto.yml, add the file under the right part in book.chapters. Order matters: list items in the order you want them in the TOC:

    chapters:
  - part: "Reference"
    chapters:
      - QUESTION-FORMAT-0.0.1.qmd
      - SETUP-0.0.1-v001.qmd
      - My-New-Doc.qmd   # new file

  4. When there are multiple versions of the same document (e.g. PRD-0.0.1-v003.qmd, PRD-0.0.1-v002.qmd, PRD-0.0.1-v001.qmd), list them with the most recent version at the top in _quarto.yml. So the newest PRD comes first, then older ones:

    - part: "PRD"
  chapters:
    - PRD-0.0.1-v003.qmd   # most recent first
    - PRD-0.0.1-v002.qmd
    - PRD-0.0.1-v001.qmd

  5. Run quarto render docs (or quarto render from inside docs/) again.

To remove a chapter, delete or comment out its entry in _quarto.yml and re-render.

4.6 Summary

Task Command (from project root)
Build the book quarto render docs
Preview live quarto preview docs
View the book Open docs/_book/index.html in a browser
